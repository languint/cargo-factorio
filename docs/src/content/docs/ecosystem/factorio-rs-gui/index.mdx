`factorio-rs-gui` is an **ecosystem** crate: reactive builders, `state!` hooks,
and mount/install helpers on top of factorio-rs.

## Add the dependency

```toml
[dependencies]
factorio-rs-gui = { git = "https://github.com/languint/factorio-rs", path = "ecosystem/factorio-rs-gui" }
```

Path/git pins match how the monorepo examples consume it today. Prefer the
published crate once it is on crates.io.

## Design notes

- Builders produce a concrete `Widget` tree.
- `state!(init)` allocates a hook slot that survives rebuilds (per root name).
- `mount` / `install` own the Factorio lifecycle glue and wire GUI events via
  `script.on_event` (no manual GUI event stubs).
- Prefer `factorio_rs_gui::shared::...` paths so Factorio `require`s resolve.
